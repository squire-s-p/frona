# shadcn/ui Knowledge for Dashboard App

Цей скіл містить специфічну інформацію про використання shadcn/ui у поточному проєкті.

## Контекст Проєкту
- **Фреймворк**: Next.js (App Router)
- **Tailwind CSS**: Версія 4 (v4)
- **Стиль**: `new-york`
- **Базовий колір**: `gray`
- **Іконки**: `lucide-react`
- **Коренева папка CSS**: `src/app/globals.css`

## Налаштування Aliases
При генерації коду завжди використовуй ці шляхи:
- Компоненти: `@/components`
- UI елементи: `@/components/ui`
- Утиліти: `@/lib/utils`
- Бібліотеки: `@/lib`
- Хуки: `@/hooks`

## Рекомендації щодо розробки
1. **Композиція**: Завжди перевіряй наявність компонентів у `src/components/ui`, перш ніж пропонувати їх встановлення.
2. **Форми**: Використовуй `react-hook-form` разом із `zod` (вони вже є в проекті).
3. **Стилізація**: Для Tailwind v4 використовуй сучасні класи та нові можливості варіацій компонентів.
4. **Анімації**: Для складних переходів використовуй `framer-motion` (якщо він є) або вбудовані утиліти Tailwind.

## Список команд для користувача
Якщо мені потрібно додати новий компонент, я маю попросити користувача виконати:
`npx shadcn add [component-name]`
